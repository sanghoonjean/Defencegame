# AGENTS.md — MakeDefence

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **게임 이름** | MakeDefence |
| **장르** | 타워 디펜스 + RTS (고정 배치형) |
| **세계관** | SF |
| **엔진** | Unity 2022.3.62f3 LTS |
| **렌더 파이프라인** | URP 2D |
| **입력 시스템** | New Input System |
| **플랫폼** | PC 우선 (모바일 확장 가능) |
| **기준 해상도** | 1920x1080 (16:9) |
| **아트 스타일** | 픽셀 아트 |
| **PPU** | 32 (32x32px 기준) |

### 핵심 게임플레이 루프

```
웨이브 반복 도전
  → 큐브(자원) 획득 (가중치 랜덤 드롭)
  → 타워 생성 / 아이템 크래프팅
  → 준비 완료 시 라스트 보스 도전 (플레이어 선택)
    → 처치 = 승리
    → 실패 = 패배
```

---

## 2. 자원 시스템 (큐브)

웨이브 클리어 시 가중치 기반 랜덤 드롭. 5종류.

| 큐브 | 역할 |
|------|------|
| **하위 큐브** | 타워 생성 + 아이템 옵션 전체 리롤 |
| **상위 큐브** | 아이템 옵션 1개 추가 |
| **최상위 큐브** | 아이템 옵션 1개 랜덤 제거 → 상위 옵션으로 교체 |
| **삭제 큐브** | 아이템 옵션 1개 랜덤 제거 |
| **복제형 큐브** | 타워 아이템 복제 |

---

## 3. 아이템 시스템

- 유닛(타워)별 인벤토리 슬롯: **3개** (추후 확장 가능)
- 아이템 옵션 수: **최소 1개 ~ 최대 6개**
- 옵션 종류: 공격력, 공격속도, 공격범위, 스턴확률 등 (옵션 테이블 추후 확정)

## 4. 웨이브 시스템

### 기본 규칙

| 항목 | 내용 |
|------|------|
| 웨이브 생성 | 수동 (+ 자동 생성 토글 on/off) |
| 플레이어 체력 | 기본 100, 웨이브 시작 시 초기화 |
| 체력 감소 조건 | 적이 기지를 통과할 때 |
| 보상 조건 | 클리어 시 1회 지급 / 체력 0 = 실패 = 보상 없음 |

### 난이도

- **1~16단계**, 플레이어 직접 선택
- 현재 단계 클리어 시 다음 단계 해금
- 단계당 적 스탯: HP/방어력 **+5%**, 이동속도 **+2%**

### 웨이브당 적 수

| 단계 | 적 수 |
|------|-------|
| 1~4  | 15마리 |
| 5~8  | 20마리 |
| 9~12 | 25마리 |
| 13~16| 30마리 |

### 적 등급 비율 공식

```
Magic  비율 = min(stage * 2%, 35%)
Rare   비율 = max(0, (stage - 4) * 3%)
Unique      = stage >= 10 ? 1마리 고정 : 0
Normal      = 나머지 (100% - Magic% - Rare%)
```

단계별 비율 예시:

| 단계 | Normal | Magic | Rare | Unique |
|------|--------|-------|------|--------|
| 1    | 98%    | 2%    | 0%   | 0      |
| 4    | 92%    | 8%    | 0%   | 0      |
| 8    | 72%    | 16%   | 12%  | 0      |
| 10   | 나머지  | 20%   | 18%  | 1마리  |
| 12   | 나머지  | 24%   | 24%  | 1마리  |
| 16   | 32%    | 32%   | 36%  | 1마리  |

### 적 등급별 스탯

- 등급이 오를수록 이전 등급 대비 HP/방어력 **+5%**
- **Unique**: 미니보스급, 10단계 이상 웨이브당 1마리 고정 등장

| 등급 | 비고 |
|------|------|
| Normal | 기준 스탯 |
| Magic | Normal 대비 HP/방어력 +5% |
| Rare | Magic 대비 HP/방어력 +5% |
| Unique | Rare 대비 HP/방어력 +5% + 미니보스 |

### 승패 조건

| 조건 | 결과 |
|------|------|
| 라스트 보스 처치 (플레이어 직접 도전) | 승리 |
| 라스트 보스 도전 실패 | 패배 |

├── Scripts/

## 6. 코딩 컨벤션 (C#)

- 클래스/메서드/프로퍼티: `PascalCase`
- private 필드: `_camelCase`
- 상수: `UPPER_SNAKE_CASE`
- `MonoBehaviour`는 View/Controller 역할만, 순수 로직은 일반 C# 클래스로 분리
- Coroutine보다 `async/await` 선호 (단, Unity 생명주기 연동 시 Coroutine 허용)
- DI: 수동 의존성 주입 (외부 DI 프레임워크 미사용)

## 7. Claude Code 작업 규칙 (Unity 특화)

- `.meta` 파일 절대 수정 금지
- `.unity` 씬 파일, `.prefab` 파일 직접 편집 금지
- `ScriptableObject` 인스턴스 생성은 에디터에서 사용자가 직접 수행 (Claude는 클래스 작성만)
- 컴파일 검증은 사용자가 Unity 에디터에서 수행
- 신규 패키지 추가 필요 시 코드 작성 전 사용자에게 먼저 확인

## 8. 테스트 정책

- Unity Test Framework (EditMode) 사용
- 순수 C# 로직은 반드시 테스트 작성 대상:
  - 큐브 드롭 가중치 계산
  - 웨이브 등급 비율 공식
  - 아이템 옵션 연산 (추가/제거/리롤)
  - 적 스탯 단계별 계산
- `MonoBehaviour` 의존 로직은 테스트 제외 (PlayMode 테스트는 선택)
