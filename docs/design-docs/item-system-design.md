# Item System Design — MakeDefence

## 1. 아이템 기본 구조

- 타워당 아이템 슬롯 **최대 3개** (모두 하위 큐브로 해금)
- 아이템 등급 없음 — 옵션 수(1~6개)로만 차별화
- 옵션은 큐브를 사용해 추가/제거/리롤/교체

```
타워 아이템 슬롯
├── 1번째 슬롯 (하위 큐브 10개로 해금)
├── 2번째 슬롯 (하위 큐브 20개로 해금)
└── 3번째 슬롯 (하위 큐브 30개로 해금)
```

---

## 2. 아이템 슬롯 해금

| 슬롯 | 해금 비용 |
|------|-----------|
| 1번째 슬롯 | 하위 큐브 10개 |
| 2번째 슬롯 | 하위 큐브 20개 |
| 3번째 슬롯 | 하위 큐브 30개 |

---

## 3. 아이템 옵션 테이블 (9종)

| 옵션 | 효과 |
|------|------|
| **공격력** | 타워 기본 공격력 증가 |
| **공격속도** | 공격 주기 감소 (빠르게) |
| **공격범위** | 타워 공격 사거리 증가 |
| **스턴확률** | 적 기절 확률 증가 |
| **치명타 확률** | 치명타 발생 확률 증가 |
| **치명타 데미지** | 치명타 시 추가 데미지 증가 |
| **관통력** | 적 방어력 무시 % |
| **스킬 쿨타임 감소** | 공격 스킬 쿨타임 감소 |
| **하위 큐브 드롭률** | 적 처치 시 하위 큐브 드롭 확률 증가 |

- 옵션 수: 아이템당 최소 1개 ~ 최대 6개
- 하나의 아이템에 같은 옵션 중복 불가

---

## 4. 큐브 효과 (아이템 크래프팅)

| 큐브 | 효과 | 비고 |
|------|------|------|
| **하위 큐브** | 아이템 옵션 전체 리롤 | 옵션 수 유지, 종류만 재랜덤 |
| **상위 큐브** | 아이템 옵션 1개 추가 | 최대 6개 초과 불가 |
| **최상위 큐브** | 옵션 1개 랜덤 제거 → 상위 옵션으로 교체 | |
| **삭제 큐브** | 옵션 1개 랜덤 제거 | 최소 1개 미만으로 감소 불가 |
| **복제형 큐브** | 아이템 복제 | 다른 슬롯에 동일 아이템 생성 |

---

## 5. 아이템 획득 흐름

```
타워 생성
  → 아이템 슬롯 해금 (하위 큐브 소모)
  → 슬롯에 기본 아이템 생성 (옵션 1개 랜덤 부여)
  → 큐브로 옵션 크래프팅
      상위 큐브 → 옵션 추가 (최대 6개)
      하위 큐브 → 전체 리롤
      최상위 큐브 → 상위 옵션으로 교체
      삭제 큐브 → 옵션 제거
      복제형 큐브 → 아이템 복제
```

---

## 6. 미확정 항목

- [ ] 각 옵션별 수치 범위 (예: 공격력 +5% ~ +30%)
- [ ] 상위 옵션의 정의 (최상위 큐브 사용 시 어떤 옵션이 상위 옵션인가)
- [ ] 리롤 시 옵션 수 변동 여부 (현재: 옵션 수 유지)
- [ ] 복제형 큐브 사용 시 슬롯이 없으면 어떻게 처리할지
- [ ] 속성 데미지 옵션 추가 가능성 (추후 검토)
