# 기능 스펙: UISystem

## 개요
게임의 모든 UI 화면과 HUD를 관리한다.
GameStateSystem 상태 변화에 따라 씬/패널을 전환한다.

---

## 기능 요구사항

### 1. HUD (게임 화면 상시 표시)

| 요소 | 위치 | 동작 |
|------|------|------|
| 플레이어 HP | 상단 좌측 | 실시간 갱신 |
| 보유 큐브 수량 (5종) | 상단 우측 | 실시간 갱신 |
| 현재 웨이브 단계 | 상단 중앙 | 단계 변경 시 갱신 |
| 웨이브 시작 버튼 | 하단 중앙 | 웨이브 진행 중 비활성화 |
| 자동 웨이브 토글 | 웨이브 버튼 옆 | on/off 전환 |
| 상점 버튼 | 하단 우측 | 상점 패널 열기 |
| 보스 도전 버튼 | 하단 우측 | 웨이브 진행 중 비활성화 |

### 2. 씬별 UI

**메인 메뉴**
- 타이틀, 게임 시작, 설정, 종료 버튼

**게임 화면**
- HUD + 맵
- 타워 클릭 시 타워 정보 패널 팝업
  - 장착 스킬 / 보조 옵션 / 아이템 슬롯 표시
  - 슬롯 해금 버튼 (큐브 비용 표시)
  - 큐브 사용 버튼

**웨이브 결과**
- 클리어/실패 텍스트
- 획득 큐브 목록 및 수량
- 확인 버튼 (게임 화면으로 복귀)

**상점**
- 공격 스킬 목록 + 구매 버튼 (하위 큐브 1개)
- 보조 옵션 목록 + 구매 버튼 (하위 큐브 1개)
- 현재 보유 큐브 수량 표시
- 닫기 버튼

**보스 도전 확인**
- 보스 스탯 정보 표시
- 도전 / 취소 버튼

**승리 (Victory)**
- 승리 텍스트
- 메인 메뉴로 / 재시작 버튼

**패배 (Defeat)**
- 패배 텍스트
- 메인 메뉴로 / 재시작 버튼

### 3. UI 공통 원칙
- 모든 수치(큐브 수량, 옵션 확률 등) 투명하게 표시
- 웨이브 중에도 인벤토리/상점 접근 가능
- 버튼 비용 부족 시 시각적으로 비활성화 표시

---

## 인터페이스

```csharp
public class UISystem
{
    public void ShowScreen(UIScreen screen);
    public void HideScreen(UIScreen screen);
    public void ShowTowerPanel(Tower tower);
    public void HideTowerPanel();
    public void UpdateHUD();
}

public enum UIScreen
{
    MainMenu, Game, WaveResult, Shop,
    BossConfirm, Victory, Defeat
}
```
