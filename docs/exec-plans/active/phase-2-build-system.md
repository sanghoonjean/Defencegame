# 실행 계획: Phase 2 — 빌드 시스템

## 목표
큐브 획득/소모, 아이템 크래프팅, 스킬 시스템, 상점을 구현해
PoE 스타일의 빌드 실험 루프를 완성한다.

## 선행 조건
- Phase 1 완료

---

## 작업 목록

### 1. CubeSystem
- [ ] 5종 큐브 수량 관리
- [ ] TryConsume() / Add() 구현
- [ ] 웨이브 클리어 시 드롭 (가중치 랜덤)
- [ ] OnCubeChanged 이벤트 발행

### 2. ItemSystem
- [ ] ItemData, ItemOption 클래스 구현
- [ ] ItemOptionType enum (9종)
- [ ] 슬롯 해금 (하위 큐브 10/20/30개)
- [ ] 슬롯 해금 시 랜덤 옵션 1개 부여
- [ ] 하위 큐브 → 옵션 전체 리롤
- [ ] 상위 큐브 → 옵션 1개 추가 (최대 6개)
- [ ] 최상위 큐브 → 옵션 제거 후 상위 교체
- [ ] 삭제 큐브 → 옵션 1개 제거 (최소 1개)
- [ ] 복제형 큐브 → 아이템 복제
- [ ] 옵션 변경 시 TowerSystem.RefreshStats() 호출

### 3. TowerSystem 스킬 확장
- [ ] SkillData ScriptableObject (5종 스킬)
- [ ] 공격 스킬 슬롯 구현
- [ ] 보조 옵션 슬롯 해금 (상위 큐브 5/10/15개)
- [ ] SupportOptionData ScriptableObject (13종)
- [ ] 스킬별 공격 방식 구현
  - [ ] 정밀 화살 (단일 타겟)
  - [ ] 화염구 (범위 폭발)
  - [ ] 마비 마법 (스턴/감속)
  - [ ] 번개 창 (직선 관통)
  - [ ] 독 구름 (DOT 범위)

### 4. ShopSystem
- [ ] 스킬/보조 옵션 구매 (하위 큐브 1개)
- [ ] 보유 목록 관리
- [ ] 큐브 부족 시 구매 불가

### 5. InventorySystem
- [ ] 타워별 인벤토리 (스킬 + 보조 옵션 + 아이템 슬롯) 통합 관리

## 완료 기준
- 웨이브 클리어 시 큐브 드롭
- 큐브로 아이템 슬롯 해금 및 옵션 크래프팅
- 상점에서 스킬/보조 옵션 구매
- 타워에 스킬 장착 시 해당 방식으로 공격
- 보조 옵션 장착 시 타워 스탯 반영
